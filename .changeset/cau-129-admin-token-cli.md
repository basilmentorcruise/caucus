CAU-129: add `caucus token` admin CLI wrapping the existing CAU-20 loopback issuer routes — `caucus token mint --owner <o> --agent <a>`, `caucus token revoke <digest>`, and `caucus token rotate <digest> --owner <o> --agent <a>` (a bare value targets one digest; `agent:<id>` targets every dynamic token for an agent id). This is an ergonomics wrapper over the same loopback admin surface an operator could hit with `curl` — **no new server capability and no new trust boundary**: the boundary is still the loopback bind plus the `CAUCUS_ADMIN_TOKEN` credential the backbone already checks. The admin token is read from the environment ONLY (`CAUCUS_ADMIN_TOKEN`) — never a flag (`--admin-token` is rejected) and never echoed (ADR-C12); the backbone URL is read from `CAUCUS_URL` (default `http://127.0.0.1:4747`). A minted/rotated token is printed once to stdout with a "copy now — not re-readable" warning and is written to no log/history file. Errors are actionable and value-free (admin token unset, control surface disabled / wrong token / non-loopback `401`, connection refused, network failure) and never echo any token bytes; every failure exits non-zero. No ADR or schema change.
